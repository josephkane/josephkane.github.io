Homebrew Buddy

One of my favorite hobbies is brewing my own beer. After a year of doing small, one gallon batches on my stovetop at home, I decided to go all in and make a full 6-gallon all-grain (no malt extract) rig. Calculating the recipes became quite a time-consuming task, so when it came time for my frontend capstone at NSS, I decided I would build an app that would calculate and store all of my homemade recipes.

Homebrew Buddy is built with AngularJs, Bootstrap, and Firebase. Ingredient-specific data was provided via the BreweryDB API. I found myself falling madly in love with AngularJs mainly for its two-way binding and routing capabilities. Also, I wanted to get used to Flexbox in CSS and found it incredibly helpful in setting up a moderately-responsive display section using very little code.

I actually use this app on a regular basis, so I have a decent list going for v2. It includes a more in-depth form validation system, adding the option to print out a recipe, moving user authentication to OAuth, and expanding the database to include "Brew Sessions," which will allow a user to keep track of their numbers each time they brew a given recipe.

A-meow-zon

This was a group project we were assigned during the backend portion of NSS. We were tasked with creating an Amazon-like ordering system for the command line. A user can sign in/register, add different payment types, shop for different products (in our case famous felines), buy them, and also print out a popularity report of all products.

All user, payment, and product data was serialized to a .txt file using Python's pickle package. Once we finished the project, we were immediately tasked with converting it to a SQLite-driven application. Switch storage initially proved more difficult than we thought, but in the end it really helped to cement all the concepts we were being taught.

One of the stretch goals my group worked on was implementing Python's curses package into our UI. It definitely made debugging a lot less straightforward (no print() or input() statements), but in the end it looked great and really set our project apart.

Lexicon Artitst

We were tasked with creating a language analysis program that read a list of phrases from a JSON file, analyzed each individual phrase based on 4 categories (behavior, sentiment, domain, and word/character tokenization).

One of the goas of this project was to learn learn the process and benefit of test-driven development. One of the major requirements was that all core logic must be tested and that all tests must be written prior to any implementation code. My group found a really helpful Python package called coverage that showed which parts of your code weren't tested properly.

Gauntlet

Gauntlet was a project in which we were tasked with re-creating the classic turn-based RPG as a means of mastering Javascript's prototypal inheritance concept. After entering their name, a user can select the species of their fighter. The species dictates (through inheritance) which classes the user has access to, which in turn (again through inheritance) dictates which weapons they have access to.

The enemy is chosen through a random enemy generator, which constructs an opponent with the same inheritance chains as the user creation logic, with randomized species/class/and weapon selections. We made that same logic available to the user via the "random" button on the landing page.
